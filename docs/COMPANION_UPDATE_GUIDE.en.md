# Updating the ChatGPT Companion and Antigravity Pipeline Together

## Update the companion when

- task framing changes;
- review/audit behavior changes;
- ChatGPT vs Antigravity boundaries change;
- evidence review policy changes;
- user-facing explanations change;
- lessons are extracted from real projects.

## Update the pipeline repository when

- README or public docs change;
- project template changes;
- workflow/rule/hook/script changes;
- validators change;
- GitHub publication scripts change;
- package/release files change.

## Update active projects only when

- the current phase is complete;
- tests/builds are green;
- audit confirms a safe window;
- a migration plan exists.

## Never

- assume companion updates automatically update Antigravity;
- assume repo docs automatically migrate H10;
- change pipeline runtime in the middle of active feature work;
- push framework patches without validation.
