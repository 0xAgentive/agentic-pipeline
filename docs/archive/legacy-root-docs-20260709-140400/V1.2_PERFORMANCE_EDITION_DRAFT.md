# Agentic Pipeline v1.2 Performance Edition - Draft

Status: draft only. Do not implement as part of v1.1.1b-r4.

## Purpose

v1.2 is intended to improve runtime efficiency without weakening the phase-gated safety model.

## Candidate themes

1. Compiled runtime: keep the full playbook as reference, but generate smaller workflow/rule runtime files.
2. Hard reference validator evolution: all referenced local files must exist.
3. Metrics: optional `.agy/RUN_METRICS.ndjson` for duration, checks, failures, and repair loops.
4. Eval suite: golden tasks for phase jumping, missing evidence, fastpatch denial, and shipcheck refusal.
5. Progressive disclosure: move long specialized procedures into narrow skills.
6. Tool-surface budget: per-workflow MCP/tool allowlists.
7. Read-only parallel audit lanes: no concurrent writers.
8. Optional prompt/cache layout: only if the executor exposes usage/cached-token metrics.

## Non-goals

- No autonomous merge loop.
- No write-capable MCP by default.
- No multi-agent maker/checker as a substitute for deterministic tests.
- No `/phasebatch` default.
