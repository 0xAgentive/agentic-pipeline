# v1.2.0 Accepted Decisions

## Accepted

1. Release name:
   `Agentic Pipeline v1.2.0 — Product Evidence Control Plane`.

2. r4/r4b stabilization is mandatory before v1.2 implementation.

3. Full playbook becomes reference-only.
   Hot-path runtime is generated/compiled and self-contained.

4. Product Contract Layer is required:
   - `.agy/PRODUCT_CONTRACT.json`
   - `.agy/REQUIREMENTS_DELTA.ndjson`
   - stale shipcheck blocked by unresolved requirement deltas.

5. Artifact Delivery Contract is required:
   - material phases create evidence archive;
   - artifact manifests include path, SHA-256, size, kind, purpose;
   - final answers must print artifact paths and checksums;
   - missing required artifacts block audit/shipcheck.

6. Evidence and metrics are machine-readable:
   - `.agy/evidence.ndjson`
   - `.agy/RUN_METRICS.ndjson`
   - `.agy/ARTIFACT_INDEX.ndjson`

7. VisualQA / ReportQA / SecurityQA become first-class gates.

8. Skills use progressive disclosure.
   Heavy domain knowledge goes into narrow skills, not root AGENTS.md.

9. Hooks have explicit mode:
   - manual
   - advisory
   - enforcing

10. `/triage` is read-only JSON classifier only.

11. `/parallel-audit` is read-only lanes only.
   No concurrent writes.

12. `shipcheck` returns only `SHIP` or `NO-SHIP`.
   `SHIP` requires deterministic evidence.

## Rejected or deferred

1. Mandatory API runtime.
2. Mandatory prompt caching.
3. Write-capable parallel agents.
4. Automatic GitHub publish/deploy.
5. Mandatory Codebase Memory, Sourcegraph, or MCP.
6. Hidden runtime brain larger than the playbook.
7. Silent durable-rule mutation from `/lessons`.
8. Model prose as verification.
9. Removing gates to improve speed.
10. Provider-specific eval dependency.
