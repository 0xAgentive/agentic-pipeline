# v1.1.1b Productivity Guards

This patch improves productivity without adding autonomous loop engineering or multi-agent routing.

## Stop limits

Agents must not run open-ended repair loops.

Default limits:

- `/nextphase`: one targeted repair pass after the first failed check.
- `/fixcritical`: two targeted repair passes for the named blocker.
- `/fastpatch`: no repair loop; if the gate fails, stop.
- `/githubsync`: no source edits; only deterministic git/gh commands.

If the limit is reached, stop and set the next safe command to `/auditphase` or human review.

## Fastpatch content guard

`/fastpatch` is not authorized by model prose.

It is allowed only when:

    powershell -NoProfile -ExecutionPolicy Bypass -File .\scripts\Test-FastPatchAllowed.ps1

exits with code 0.

The gate must check both changed paths and added diff lines. A file being allowlisted is not enough if its added lines introduce backend coupling, network/storage calls, unsafe DOM APIs, dynamic evaluation, or Node/system APIs.

## Semantic verification

For critical projects, semantic readiness is verified by deterministic tests and exit codes, not by model review.

Negative-control or fault-injection checks should be project-specific. Do not add a generic AI-driven fault-injection workflow by default.

## GitHub publication evidence

GitHub publication must use deterministic `git` and `gh` commands. GitHub MCP is not required.

If `.agy/EVIDENCE_LOG.md` exists, GitHub sync scripts may append compact machine-produced evidence:

- timestamp;
- command;
- branch;
- commit hash;
- remote URL;
- result.

Do not stage `.agy/EVIDENCE_LOG.md` unless the project explicitly tracks local evidence.


## Package hardening addendum

The release package must pass hard validation before publication.

Required checks:

- referenced script paths exist;
- PowerShell files parse;
- JSON state files parse;
- template hooks are not no-op placeholders;
- `.cbmignore` includes the canonical context exclusions;
- GitHub Actions validate workflow uses read-only token permissions.

Do not treat `validate-package.sh` alone as sufficient for public releases.
