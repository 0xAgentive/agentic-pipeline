# Runtime Truth — Scoped Assurance

A capability is active only when its files, scripts, parameters and validators exist and pass.

Runtime truth is scoped:

- product execution authority is distinct from release/publication authority;
- a legacy governance defect may degrade release authority without invalidating an owner-approved product work item;
- expected executor edits do not invalidate their own work-item authorization;
- checkout to another branch, owner-goal change, outside-scope external edits, runtime migration or a new material risk invalidate the execution lease;
- `RUN_RESULT.json` supersedes stale walkthrough counts and summaries;
- hashes are trust-boundary metadata, not a daily human approval mechanism.
