# v1.2 Runtime Truth

A documented capability is considered active only when its files, scripts, parameters and validators exist and pass.

Examples:

- workflow command flags must exist in the referenced script;
- root and template safety scripts must have equivalent behavior;
- active hooks require non-empty verified hook configuration;
- v1.2 project status requires `agentic_pipeline_version: 1.2.0` and a recorded framework source commit;
- evidence gates require the corresponding files and blocking validator behavior;
- legacy installers must not downgrade canonical runtime files.

When a claim and runtime differ, runtime evidence wins and the state is BLOCKED until repaired.
