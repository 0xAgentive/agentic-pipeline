# Быстрый старт

## Что это?

Agentic Pipeline — это простая инструкция, как работать с агентом Antigravity на реальных проектах.

Он защищает от типовой проблемы:

```text
агент начал писать код
агент изменил слишком много
агент написал “готово”
тесты или реальность продукта не совпали
потом приходится вручную разбирать, что произошло
```

Пайплайн делает цикл проще:

```text
понять, что строим
описать ТЗ
составить план
реализовать только следующую фазу
проверить
записать evidence
остановиться
```

## Какую команду вызывать сейчас?

Если есть только идея:

```text
/specdoc
```

Если ТЗ уже есть, но нет плана:

```text
/planonly
```

Если проект уже существует и непонятно, что в нём правда:

```text
/auditphase
```

Если план есть и аудит чистый:

```text
/nextphase

Implement only the next planned phase. Stop after verification and checkpoint.
```

Если менялся интерфейс:

```text
/visualqa
```

Если есть приватные данные, экспорт, работа с файлами, MCP, credentials или безопасность:

```text
/securityaudit
```

Если нужно понять, готов ли проект к релизу:

```text
/shipcheck
```

Если первая публикация на GitHub:

```text
/githubprepare
/githubsync
```

Если репозиторий уже есть:

```text
/githubsync
```

## Как не сломать действующий проект

Не обновляй пайплайн, пока агент прямо сейчас меняет код продукта. Сначала закончи текущую фазу. Потом попроси read-only audit состояния пайплайна. Миграцию делай только отдельной фазой, когда workspace чистый.
